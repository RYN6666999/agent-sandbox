# 協作模式分析：Subagent vs WolfPack vs 跨視窗

> 目標：「一件事用最快、最少手動介入的方式完成。」

---

## 核心觀念

**同一視窗內的 Subagent / WolfPack 是「父子關係」—— 我的延伸工具。**
**不同視窗之間是「平行關係」—— 各自獨立，不需要連動。**

跨視窗不是派工系統。要派工，WolfPack 在同一窗就解決了，不需要你切視窗。

---

## 同一視窗：父子關係（Subagent / WolfPack）

我就是主 agent，Subagent 和 WolfPack 是我的延伸工具 —
幫我做事、結果回來給我整合。

```
┌──────────────────────┐
│  我（Planner 主 agent）│
│  ┌────────────────┐   │
│  │ Subagent /     │   │
│  │ WolfPack       │   │
│  │ = 我的延伸工具  │   │
│  └────────────────┘   │
└──────────────────────┘
```

### Subagent — 單一子代理，串接任務

一句話 → spawn 一個子代理 → 做完回報 → 我接著用。

**適合**：單一任務、步驟有先後、結果要回來整合
**關係**：我叫它去做，它做完回來

```
你：「push 上 GitHub」
我 → spawn coder subagent → git add/commit/push → 回報結果給你
（零手動切換）
```

### WolfPack — 多個子代理，平行任務

一次 spawn 多個子代理同時跑，各自做不同的事。

**適合**：多個獨立任務，彼此不依賴
**關係**：我叫它們各自去做，全部回來後統整

```
WolfPack →
  ├── Agent A: git commit + push
  ├── Agent B: pytest tests/
  └── Agent C: 寫 release note
總時間 = 最慢的那個
```

### Subagent vs WolfPack 核心取捨

| | Subagent | WolfPack |
|---|---|---|
| 同時跑幾個 | 1 個 | N 個（最多 20） |
| 總時間 | 任務數 × 平均時間 | 最慢的那個任務 |
| 任務依賴 | 可以有（B 等 A） | **不能有**（各自獨立） |
| 適合場景 | 單一步驟鏈 | 多條獨立工作線 |

一句話記：**Subagent 串接、WolfPack 並聯**。

## 不同視窗：平行關係（兩個獨立 Scream session）

```
┌──────────────────────┐      ┌──────────────────────┐
│  視窗 A（Planner）    │      │  視窗 B（另一條線）   │
│  ┌────────────────┐   │      │  ┌────────────────┐   │
│  │ Subagent /     │   │      │  │ Subagent /     │   │
│  │ WolfPack       │   │      │  │ WolfPack       │   │
│  └────────────────┘   │      │  └────────────────┘   │
│                       │      │                       │
│  做自己的事（開發A功能）│      │  做自己的事（研究B主題）│
└──────────────────────┘      └──────────────────────┘
           │                            │
           └──────── 腦庫（偶爾講一聲）──┘
```

兩個視窗是**平行關係**，各有完整 context、記憶、能力，各自獨立運作。

**跨視窗不是派工**。沒有「Planner 派工給 Executor」這回事。
- 如果要派工 → WolfPack 在同一窗解決
- 如果要切視窗手動下指令 → 不如直接在同一窗做完

### 腦庫（AgentOS brain API）的真正用途

不是任務調度系統，只是**偶爾喊一聲的通訊板**：

- 視窗 A：「我這邊 agent-sandbox 開發完了」（寫 brain）
- 視窗 B：「收到，那我開始接」（讀 brain）
- 不用連動、不用協調、各自專心

### 跨視窗唯一真正的價值

**同一時間平行推進兩條獨立路線，互不打擾。**
例如：

- 視窗 A：開發 agent-sandbox 功能
- 視窗 B：研究 GASP skill / 讀另一份 codebase
- 各做各的，腦庫只是偶爾「我好了」、「你那邊呢」

---

## 決策樹（更新版）

```
任務來了
│
├── 要平行推進兩條獨立路線？
│   └── YES → 開另一視窗，各做各的，腦庫只是偶爾通訊
│
├── 可拆成多個獨立子任務？
│   └── YES → WolfPack（平行加速）
│
├── 單一任務，步驟明確？
│   └── YES → Subagent
│
└── 很簡單（2-3 個指令）？
    └── 直接做掉，不用派工
```

---

## 具體案例對照

| 任務 | 最佳模式 | 原因 |
|------|---------|------|
| git push 到 GitHub | **直接做／Subagent** | 單一步驟 |
| push + pytest + 更新文件 | **WolfPack** | 三個獨立任務可平行 |
| 同一時間開發 + 研究另一主題 | **跨視窗平行** | 兩條獨立路線 |
| 寫一段小程式碼 | **Subagent／直接做** | 單一任務 |
| 搜尋 codebase + 改多檔 | **WolfPack** | Grep 多處 + 改不同檔案可平行 |
| 「你好」 | 直接回答 | 不需要任何派工 |

---

## 工作規範

1. **同一窗內能解決的，絕不開另一窗**
2. **Subagent 串接、WolfPack 並聯、太簡單的直接做**
3. **跨視窗 = 平行獨立路線，不是派工**
4. **腦庫只是偶爾通訊板，不是任務調度系統**
