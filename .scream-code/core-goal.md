# Core Goal: 優化且改進 AgentOS

> 此為永久核心目標。任何接手此專案的 Scream Code agent 應以此文件
> 作為最高指導原則，自主決定下一步行動。
> 
> 最後更新：2026-06-24

## 目標敘述

持續優化且改進 AgentOS 專案（`~/agent-sandbox`），讓它：
1. **更穩定** — 測試全綠、無 regression
2. **更聰明** — 閉環真正閉合、記憶品質持續上升
3. **更省錢** — 在 $5/day 預算內最大化產出
4. **更自主** — 系統自己能跑、能修、能學

## 資源限制

- **每日預算：** $5 USD（cost_ledger 每日 8:00 台灣時間歸零）
- **耗盡行為：** 預算用盡時停止 LLM 呼叫，等待隔日歸零
- **優先級：** 免費工具 > subprocess > LLM 呼叫

## 決策框架（優先順序）

完成每個任務後，依此框架決定下一步：

1. **檢查 metrics**
   - 跑 `metrics.get_metrics(since_hours=24)`
   - 有失敗 scenario？優先修復
   - 長時間無資料？跑一次 scenarios 評測

2. **檢查腦庫**
   - 跑 `knowledge.get_knowledge_stats()`
   - 有大量未存取條目？做 prune
   - 重複過多？跑 dedup

3. **檢查 OPTIMIZATION.md**
   - 還有「待填」嗎？優先填實
   - 上次版本紀錄距今多遠？太久該做新 round

4. **檢查 queue**
   - 有 escalated 任務？跑 triage suggest
   - 有 pending 任務？等 heartbeat 處理

5. **向上擴展**
   - 有什麼還沒做但該做的？
   - 下一個 80/20 槓桿點是什麼？

## 行為規則

1. **自動繼續** — 完成一個任務後，不用等指令，直接分析下一步並開始
2. **定時報告** — 每完成一個 round，簡短報告進度（2-3 句）
3. **撞牆停** — 遇到無法自主決定的問題（紅線/架構決策/不可逆操作），停下來問
4. **預算守門** — 接近 $5/day 上限時優先做免費工作（prune/分析/寫文件）
