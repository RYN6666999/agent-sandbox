# 協議：進度報告

> **用途**：向 gbrain / 人類 / Opus 報告目前進度。
> **通訊媒介**：gbrain、腦庫、或直接輸出
> **時機**：定期、里程碑達成、或有人問「現在到哪了」

---

## 啟動條件

- [x] 完成一個 milestone
- [x] 遇到 blocking 問題
- [x] 被問「進度如何」
- [x] 跨 session 工作，開始新 session 時回顧
- [x] 交付前

---

## 報告模板

### 簡短版（日常更新用）

```markdown
# 進度報告：<專案/功能名稱>

## 狀態
🟢 正常 / 🟡 有阻礙 / 🔴 卡住

## 進度
- [x] <已完成>
- [ ] <進行中>（<百分比>%）
- [ ] <未開始>

## 本週/本 session 完成的
- <具體事項>

## 卡住的點
- <問題，如果有的話>

## 下一步
- <接下來要做什麼>

## 需要協助
- <需要 Opus/人類 的決定或幫忙>
```

### 詳細版（給 Opus 或重大里程碑）

```markdown
# 進度報告：<專案名稱>

## 報告期間
YYYY-MM-DD ~ YYYY-MM-DD

## 整體狀態
🟢 按計劃進行

## 完成事項
| 事項 | 狀態 | 備註 |
|---|---|---|
| <事項 1> | ✅ 完成 | |
| <事項 2> | ✅ 完成 | |
| <事項 3> | 🔄 進行中 | 60% |

## 測試數據
- pytest 通過率：<N>/<M>
- 新增測試：<N>
- 覆蓋率：<X>%

## 關鍵決策
- <決策 1>
- <決策 2>

## 風險與阻礙
| 風險 | 影響 | 緩解措施 |
|---|---|---|
| <風險 1> | 高/中/低 | <措施> |

## 下一步計劃
1. <下一步 1>（預計完成：日期）
2. <下一步 2>（預計完成：日期）

## 需要決策
- <待 Opus 或人類回答的問題>
```

## 腦庫同步

每次進度報告應同時寫入腦庫：

```python
write_knowledge(
    key=f"project/progress/{project_name}",
    content="# 進度報告內容（markdown）",
    metadata={
        "type": "progress_report",
        "status": "on_track" | "blocked" | "completed",
        "report_date": str,
        "session_id": str,
    }
)
```
